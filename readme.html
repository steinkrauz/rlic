PROTOCOL

RLicServer supports following commands

  • USER <login> <os user>
  • RELOAD
  • QUIT
  • CFG [ON|OFF] <key>

USER

Server checks the given login agains its configuration and returns 'ACCESS DENIED' if the login is not allowed from client's IP or 'ACCESS GRANTED' otherwise. The 'os user' parameter is meant to be put in log.

RELOAD

Causes the server to reread its configuration. Should be used with admin utility only.

QIUT

Terminates the server. Should be used with admin utility only.

CFG

Enables or disables configuration commands.

CONFIGURATION PROTOCOL

RLicServer supports following configuration commands

  • LISTNW
  • LISTUSER <network>
  • ADDNW <network>
  • ADDUSR <network> <login>
  • REMNW <network>
  • REMUSR <network> <login>
  • SAVE
  • LOAD
The names of commands are self-explanatory.

CONNECTION

All data exchange between server and client is assumed to be encrypted. To communicate with server one should use supplied RLicReader/RLicWriter classes that are deriviates from BufferedReader and PringWriter.

FILES

  • rlicsettings.xml — contains configuration data for the server. File format is self-explaining. You can exec RLicCfgGen to get a working example if current one is badly corrupted.
  • log4j.xml — standard log4j configuration. See teh Internets for help.
